# 선착순 쿠폰 발행 시스템

## 프로젝트 상태

팀 구성 변경으로 인해 본 프로젝트는 개발이 중단되었습니다.
현재 저장소는 기존 작업 내용을 보존하고 참고하기 위한 목적으로 유지합니다.

## 프로젝트 개요

선착순 쿠폰 발행과 쿠폰을 적용한 주문 흐름을 구현하기 위한 백엔드 프로젝트입니다.

현재 주요 도메인은 다음과 같습니다.

- 상품
- 쿠폰
- 주문
- 사용자

## 현재 구현 범위

### 상품

- 상품 목록 조회
- 상품 단건 조회
- 상품 목록 페이지네이션

### 주문

- 주문 생성
- 주문 미리보기
- 주문 목록 조회
- 주문 상세 조회
- 주문 취소
- 쿠폰 적용 주문 생성
- 주문 취소 시 쿠폰 사용 상태 복구

### 사용자

- 회원가입
- 중복 이메일 검증

### 공통

- 공통 API 응답 구조
- 전역 예외 처리
- 요청 DTO 검증
- 서비스 테스트 일부 작성

## 현재 구현 기준

- 인증 구현 전까지 `userId`는 임시 query parameter를 통해 전달합니다.
- 쿠폰 도메인 구현 전까지 주문 도메인 내부의 `reference` 모델과 서비스를 통해 쿠폰 정보를 참조합니다.
- 주문 취소는 API 명세에 따라 hard delete 방식으로 구현되어 있습니다.
- 금액 계산은 `BigDecimal`을 기준으로 처리합니다.
- 목록 조회는 Spring Data `Pageable`을 사용합니다.
